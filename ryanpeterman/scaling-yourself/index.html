自我擴展

主管告訴他，他最主要無法升任主任工程師（Staff engineer）的原因在於一個關鍵差距，這些不足的地方大家都差不多
他需要提升自己的廣度，這與從 Junior 升到 Mid-level 或 Senior 有些不同
因此，這是最常見的一個阻礙讓人無法成為主任工程師
需要換個思維模式，你不再只是自己做事情，而是需要運用你的影響力去影響他人

讓我們深入探討這個想法，提升你的廣度
這代表著你不會像其他一些工程師一樣卡在 Senior 的級別
提升廣度的關鍵在於擴展誰或什麼可以讓你的行為更有影響力
這能讓你更有影響力，而不需要投資更多時間
以下是一些方法和例子

第一，影響和委派

找到一個具有影響力的問題，並指導他人如何解決這些問題
這種方式通常比你自己寫程式來解決問題更省時間
那麼，功勞是否還是你的呢？你可以想想，如果沒有你在，這件事會怎麼樣？
如果只有你才能讓這件事完成，那這些功勞就是屬於你的
因此，一些技術領導者（Tech Leads）寫很少的程式碼，但仍然非常有影響力
作者也寫了一些文章（請見原文連結），這些可能有助於你學習這個關鍵技巧

第二，分享知識

老子說：授人以魚不如授人以漁

分享知識可以讓別人更有影響力，而你不需要持續投資時間
這種方法能拓展你的廣度，視你讓多少人成長以及這些人的潛力而定
以下是一些例子：

  1. 技術分享：舉辦技術講座，讓聽眾不需花費太多時間就能學習新技術你也可以錄下講座，擴大觸及面，讓當天沒空的人也能受益
  2. 寫作：在論壇或維基百科上回答常見問題，這些回答會一直存在，持續影響他人
  3. Code Review：每次的程式碼審查都是給他人反饋的機會，讓他們寫出更好的程式碼
  4. 辦公室時間：安排專屬的辦公室時間，集中解決大家的問題

第三，建造工具

開發者工具能提升廣度，因為這些工具可以重複使用
如果你開發了一個工具，例如自動化排程、測試工具或圖像化工具，想想如何打包這些工具，讓其他人也能使用並受益
工具的影響力取決於有多少人使用它
如果你想知道更多如何打造工具並最大化其影響力，這裡有一篇文章可以參考《何時構建工具以及如何最大化其影響力》

如果你正在提升你的廣度，你會有更多的影響力，而不用花費更多時間上的投資
這些行為是持續進行的，這就是為什麼那些人能達到主任級別，因為他們持續做這些事情
如果你是比較資淺，可以試試剛剛提到的分享知識和製造工具
不要只是一味埋頭苦幹，分享知識和工具會讓你更快速地獲得更多影響力
如果你覺得這些方法很有用，可以分享給朋友，並訂閱文章！

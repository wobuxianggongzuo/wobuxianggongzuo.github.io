前言

沒錯！看到標題就知道，我又重回 Hexo 的懷抱了！！
因為我真的太喜歡 NexT 主題了
之前會換到 Hguo 是因為速度比較快，加上又有超強的人把 NexT 主題做出來
但是最後想一想還是回去用最原生的方案：Hexo + NexT
所以我在這兩個之間的轉換，遇到了一些問題，主要就是『自動化的部署 github page』時遇到的問題
造成這個問題的變數有兩個：

  1. github action 的部署方式
  2. NexT 版本以及安裝的方式

NexT 版本

版本 8.19.0 有 bug，會無法顯示內容
當我開始要用 NexT 的時候，測試了一下發現內容都無法顯示，去到 github 上面看版本剛剛更新不到一天，果不其然是版本有 bug，換回前一版 8.18.2 就沒問題了

NexT 安裝方式

NexT 的安裝方式有兩種，參照官方的 github，第一種是用 npm，第二種是直接 clone
一開始我是想用 clone 的方式，感覺可以比較好的管理版本（還是習慣 theme 資料夾裡面有東西）

clone

這個方法遇到的問題是，clone 下來的資料夾（theme/next），有包含 .git，會使得 git 在 push 的時候不會被傳到 github 上面，當然也是有對應的解法：

  • 簡單粗暴解決
    直接刪掉 NexT 中的 .git 資料夾，將主題檔當成普通檔案加入版控中，缺點是之後無法更新主題
  • submodules
    在 hexo 中使用 git submodules 管理主题
  • subtree
    Hexo主题同步

在嘗試上面上個解法的時候還是遇到一些問題，而這些問題也都沒有解，這邊只是記錄一下
最後是嘗試很多組合，在使用 npm install 的時候可以完整安裝，所以也就沒有再去深究這些問題了

單純 clone 下來後，刪除 .git 資料夾
參考連結：Hexo + GitHub Actions 部屬網站遷移全紀錄 - Jerry Yang’s Blog
部署方式

[!Warning] css error
有花時間找一下資料，覺得應該是 push 上去的資料不完整之類的

使用 submodules，也調整部署方式，直接用 github action market 裡面 hexo 最多星星的部署方式，但最後更新也是三年前了，會噴錯也是很正常的（？

參考連結：sma11black/hexo-action: Hexo CI/CD Action for automating deployment.
部署方式

[!Warning] Error: Cannot find module ‘hexo-util/lib/spawn’
這是把 NexT 改用 submodules 噴了一堆錯誤出來，完全不知道問題在哪邊

npm install

這是嘗試後的安裝方法，在家裡的電腦想要指定版本會噴錯，到公司後把資料夾重新 clone 下來就可以了，應該是有東西沒刪除乾淨
參考連結：(27) 試著學 Hexo - 奇淫技巧 - 自動化部屬(GitHub Actions) | 是 Ray 不是 Array
部署方式

[!Warning] ERROR Deployer not found: git
剛設定完，也部署成功，但網頁還是每丁每當，完全沒有任何一點點變化
點進去部署裡面看，發現有 not found git，還好只是沒安裝hexo-deployer-git ，使用 npm insatll 裝完之後就可以了～終於！！！

心得

  • 弄了半天，結果最後部署的方式，跟我最早開始用 hexo + github action 設定的幾乎一樣，看來古人（以前的我）的智慧還是不容小覷的！
  • 接下來就是要調整 UI 以及各種套件的安裝，希望都可以順順利利
  • 最初一開始設定的 github action key ，從 Hexo 到 Hugo 又到 Hexo，都還可以無痛使用，真的方便呢！
